# Beyond AI 퍼실리테이터 가이드

> **대상**: Beyond AI 프로그램 운영 퍼실리테이터
> **버전**: v1.0 | 작성: JJ Creative 교육연구소

---

## 1. 퍼실리테이터 역할

Beyond AI 퍼실리테이터는 단순한 진행자가 아닙니다. 당신은:

- **감독관**: AI와 사람이 동일한 조건에서 경쟁하도록 공정한 환경 조성
- **코치**: 참가자들이 자신의 인간적 강점을 발견하도록 촉진
- **채점자**: AI 자동 채점의 맥락을 이해하고 보너스 점수 판단
- **스토리텔러**: 각 퀘스트가 단순한 활동이 아닌 성장 경험이 되도록 의미 부여

## 2. 세션 전 준비 체크리스트

### 2주 전
- [ ] 해당 퀘스트 시나리오 선정 (A/B/C/D 중)
- [ ] 참가자 수에 맞는 팀 구성 계획
- [ ] 필요 데이터/자료 준비 (시나리오별 별도 패키지 확인)
- [ ] 플랫폼 AI 배틀 세션 설정

### 전날
- [ ] 시나리오 자료 인쇄 또는 디지털 배포 준비
- [ ] 타이머/스크린 셋업 확인
- [ ] AI 응답 미리 실행하여 결과 확인 (당일 비교용)

### 당일 30분 전
- [ ] 참가자 좌석 배치 (팀별 또는 개인)
- [ ] 채점 시트 준비
- [ ] AI 배틀 인터페이스 작동 확인
- [ ] 브리핑 자료 화면 공유 준비

## 3. 세션 진행 가이드

### 브리핑 (5분)

**원칙**: 최대한 짧고 강렬하게. 참가자들을 과제 상황 안으로 빠르게 몰입시키세요.

```
권장 브리핑 오프닝:
"여러분은 지금 [상황 설정]입니다. AI도 옆에서 동일한 과제를 받았습니다.
30분 후에 결과를 비교합니다. 시작하겠습니다."
(15초 이내)
```

**피해야 할 것**:
- 과제 방법 과도하게 설명 (자율성 침해)
- "AI보다 잘해야 한다"는 압박성 발언
- 평가 기준 너무 상세히 설명 (자연스러운 행동 방해)

### 진행 중 관찰 포인트

다음을 메모하며 채점 & 디브리핑에 활용하세요:

```
관찰 노트 양식:
[ ] 과제 시작 5분 내 팀/개인이 어떻게 접근했는가?
[ ] 어떤 팀이 예상치 못한 접근법을 취했는가?
[ ] "AI가 절대 못할 것 같은" 순간이 포착됐는가?
[ ] 막힌 팀/개인이 있었는가? 어디서?
[ ] 시간 관리는 어떠했는가?
```

### 마무리 & 디브리핑 (5-10분)

**3단계 디브리핑 구조**:

1. **결과 공개** (1-2분)
   - AI와 사람 점수 비교 화면 공개
   - 단, 점수 발표 전 먼저 "어떤 느낌이었나요?" 질문

2. **반응 나누기** (2-3분)
   - "오늘 AI보다 잘한 부분이 있다면?" (사람 관점)
   - "AI가 예상보다 잘한 부분이 있었나요?" (AI 관점)

3. **인사이트 추출** (2-3분)
   - "오늘 배운 것 한 가지를 한 문장으로?"
   - "다음 퀘스트에서 다르게 할 것은?"

## 4. 퀘스트별 운영 특이사항

### Q1 시장 분석
- 데이터 제공 방식: 퍼실리테이터가 간단한 표/차트로 사전 준비
- AI 대전 방식: AI는 미리 동일 데이터로 분석 완료 → 세션 중 결과 공개
- 주의: 참가자들이 "정답"을 찾으려 하지 말고 **관점**을 갖도록 유도

### Q4 위기 대응
- **15분 업데이트**: 퍼실리테이터가 직접 "긴급 상황 업데이트"를 공지
  - 예시: "방금 언론에서 연락이 왔습니다. 기자가 2시간 내 기사 게재 예정이라고 합니다."
- 의도: 계획 수정 능력, 유연성 평가
- AI는 업데이트 버전 응답도 별도로 준비

### Q6 프레젠테이션
- 실제 발표 진행: 참가자들이 실제로 발표함 (AI는 텍스트/슬라이드로 대체)
- 청중 역할: 퍼실리테이터 + 다른 참가자들이 임원 역할
- 발표 중 질문 1-2개 던지기 (임기응변 테스트)

### Q8 고객 응대
- 퍼실리테이터가 "화난 고객" 역할 직접 수행
- 채팅 형식: 메신저 채팅창 활용 또는 종이/화이트보드
- 강도 조절: 참가자 반응에 따라 고객 화남 정도 조절 가능
- **절대 금지**: 개인 공격성 발언, 과도한 폭언 (교육 목적 유지)

### Q12 연간 발표
- 팀별 15분 순서 배정 (참가자 수에 따라 조정)
- 전 참가자 청중으로 참여
- 발표 후 시상식 — 미리 수상자 선정 불필요, 발표 보고 현장 판단

## 5. 어려운 상황 대응 가이드

### 상황 1: "AI가 이겼을 때" 참가자 사기 저하

**대응**: 점수 결과를 발표 전에 프레임 바꾸기
```
"점수를 보기 전에, 오늘 여러분이 작성한 내용 중
AI가 절대 쓸 수 없었을 문장이 있었나요?"
```
그런 다음 점수 공개. 설령 AI가 이겼더라도 **참가자가 발휘한 인간적 강점에 집중**.

### 상황 2: 참가자가 과제를 너무 어려워하거나 아무것도 못 씀

**대응**: 방향 제시 (답 제시 X)
```
"지금 어디서 막혔어요? 처음 떠오른 것 하나만 써보세요."
"완벽하지 않아도 됩니다. 지금 생각나는 것부터."
```

### 상황 3: 참가자가 AI 결과를 사전에 검색하거나 ChatGPT를 활용

**정책 결정 (기업 계약 전 합의)**:
- 금지 정책: AI 도구 사용 금지, 위반 시 해당 퀘스트 0점
- 허용 정책: AI 도구 활용 가능, 단 "AI 도움 비율"을 공시해야 함

> **권장**: 허용 정책. "AI를 어떻게 활용하는가"도 역량입니다.

### 상황 4: 참가자들이 디브리핑에 참여하지 않음

**대응**: 1:1 질문으로 시작
```
"[이름]님, 오늘 제출하면서 어땠어요? 한 단어로요."
```
한 명이 대답하면 나머지가 따라옵니다.

## 6. 채점 가이드

### AI 자동 채점

플랫폼이 4대 축으로 자동 채점합니다. 퍼실리테이터는 이 결과를 그대로 사용하되, 다음 경우에 조정 가능합니다:

**보너스 점수 부여 가능 상황** (최대 +15점):
- 결과물에서 "AI가 절대 도달 못할 인간적 통찰"이 명확히 포착될 때
- Q6 발표에서 청중이 눈에 띄게 감동하거나 반응을 보일 때
- Q10 아이디어에서 청중 투표 1위를 달성할 때
- Q12 발표에서 임원급이 자발적으로 메모를 시작할 때

**감점 가능 상황** (최대 -10점):
- 표절 또는 AI 무단 사용이 명백할 때 (과제별 정책에 따름)
- 제출 시간을 심각하게 초과할 때 (사전 고지 없이)
